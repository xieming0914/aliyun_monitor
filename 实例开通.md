## 🧠 核心概念：什么是 CDT 公网流量？

**CDT (Cloud Data Transfer)** 是阿里云提供的一种统一流量计费模式。

* **免费额度**：开通 CDT 后，国际站账号名下的 ECS 每月享有 **220GB** 的公网流量免费额度（其中 20GB 可用于中国大陆，200GB 限制非中国内地，即**香港/新加坡等亚太热门节点完全可用**）。
* **计费逻辑 (重要)**：
  - **出入取最大 (Max In/Out)**：CDT 的计费流量通常取 **“流入流量”** 和 **“流出流量”** 中的**最大值**进行计算。
  - **举例**：如果你下载了 100GB 文件（入），上传了 10GB 文件（出），系统将扣除 **100GB** 的免费额度。
* **线路质量**：走的不是普通线路，而是阿里云优质的 BGP 线路。在香港和日本，新加坡区域，直连国内速度极快。

## 🚀 部署流程

### 第一步：开启 CDT 共享流量包 (至关重要)

如果不开启此项，流量将按昂贵的标准价格计费！
1. 登录阿里云即可。
2. 在顶部搜索框输入 **“CDT”**，点击自动弹出的 **“控制台 > 云数据传输”**。
* 或者直接访问：[https://cdt.console.alibabacloud.com/overview](https://cdt.console.alibabacloud.com/overview)
3. 进入 CDT 工作台概览页，检查页面上的“公网流量”或“云数据传输”子项。如果显示“未开通”或“未升级”，请点击右上角的 **“一键升级”** 或 **“开通服务”** 按钮。确认开通后计费模式即刻生效。

### 第二步：开通 OSS 并上传 Alpine 镜像 (极致省钱)

为了进一步降低磁盘费用和系统资源占用，我们推荐使用 **Alpine Linux** 这种超轻量级系统。我们需要先将镜像上传到阿里云 OSS，再导入为 ECS 自定义镜像。

1. **下载 Alpine 虚拟化镜像**：
   * 请直接前往本项目 Releases 页面下载特制的 ISO 镜像，或点击下方按钮直接下载：
   * 👉 [<img src="https://img.shields.io/badge/Download-alpine--virt--3.23.2-blue?style=for-the-badge&logo=download" />](https://github.com/10000ge10000/aliyun_monitor/releases/download/v1.0/alpine-virt-3.23.2-x86_64.iso)

2. **开通 OSS 服务**：
   * 控制台搜索 **“OSS”** (对象存储) 并开通。
   * **创建 Bucket**：
     * **区域**：必须与你稍后要购买 ECS 的区域一致（推荐 **中国香港** 或 **日本**）。
     * **存储类型**：标准存储。
     * **读写权限**：私有。

3. **上传与镜像导入**：
   * 进入该 Bucket 上传该 `.iso`。
   * 进入 **ECS 控制台** -> **实例与镜像** -> **镜像**，点击 **“导入镜像”**。选择刚才上传的 `.iso` 文件。操作系统选 Linux / Other Linux，系统盘大小建议设置为默认（或者取决于您后续磁盘诉求配比）。

### 第三步：创建“抢占式”实例

1. 进入 ECS 购买页面 (创建实例)。
2. **付费模式**：选择 **抢占式实例 (Pay-As-You-Go)**。
3. **地域**：选择你上传镜像的区域（与刚才 OSS 地域保持一致）。
4. **实例规格**：
   * **架构**：x86 计算。
   * **分类**：选择 **“抢占式实例” (Preemptible Instance)**。
   * **策略**：选择“无保护期”或默认策略（价格最低）。
   * **规格选择**：筛选 **2 vCPU / 0.5 GiB** 或 **1 vCPU / 1 GiB** 的入门级实例（例如 **经济e 系列** 或 `t5`/`t6` 系列）。通常这种实例的时价低至 **$0.001/小时** 甚至更低。
5. **镜像**：点击 **“自定义镜像”**，选择你在第二步导入的 Alpine。
6. **存储**：选择 **ESSD Entry** 或 **高效云盘**。此时系统盘容量 **建议 3GB 起**，以保证足以放下运行脚本的 Python 环境和其他补丁。
7. **公网 IP (核心步骤)**：
   * **分配公网 IPv4 地址**：**坚决不勾选！**
8. **安全组**：开放 SSH (22) 端口。
9. **下单**：确认配置与预估极低的费用后点创建，实际流量费将由 CDT 抵扣。

### 第四步：创建弹性公网 IP (EIP)

* 👉 **[前往 弹性公网IP 控制台](https://vpc.console.alibabacloud.com/eip)**
* 创建申请一个同地域的免费 EIP，并将其 **绑定至上一步刚申请成功的实例**，最后加入带宽包服务畅快使用即可。

*(之后您可以参考 README.md，将生成的实例使用 `install.sh` 纳管到自动化监控风控脚本中)*

## ❓ 常见问题 (FAQ)

**Q: 抢占式实例被释放了怎么办？**
A: 这是抢占式实例的特性。数据请务必通过 OSS 或其他方式实时备份（或快照）。如果实例被释放后，你需要重新走一遍购买配置流程并把新 ID 更新到监控管理中去。

**Q: 为什么账单里还是会有几分几毛钱产生？**
A: 可能是因为实例机器本身的超小额计算租金、磁盘租赁费，也可能是超过了 200G 的流量包产生了标准的抵扣费用。CDT 只免流量费，不免机器基础硬件本身的钱。

**Q: 下载文件算流量吗？（重点注意）**
A: **绝对算！** 再次重申，CDT 计费采用 **“双向取最大值 (In/Out 选大者)”** 策略。
* 如果你主要用于 **建站/跑对外服务**（主要是流出为主），则按照上行流出流量计费。
* 如果你主要用于 **作为客户端去海量下载别人资源/拉取网盘备份**（主要是流入为主），系统就会反过来向你针对下行的输入流量计费。
* **警告**：请勿抱着“下载不花钱”的心态随意发呆进行极端压测或者长期离线挂种大体积 BT，200G 额度如果满载百兆线路一晚上就能被刷爆！
